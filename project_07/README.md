# Golang 实现一个短链接服务

## 短连接的应用场景和作用

### 场景
>
> 1.营销短信
> 
> 2.URL 安全性需求
> 

### 作用
>
> 1.使得连接变得更短
> 
> 2.屏蔽原始 URL 中的参数
> 


## 实现短链接需要考虑的问题

- 短链接到底有多短，1~11位（int64最大值转成62进制也就11位）
- 短链接用什么算法，自增 ID
- 缓存问题，短链接和长链接匹配关系缓存多久，缓存的加载方式，缓存的穿透问题。
- 短链接的访问问题
  - 类似图片能否内嵌到 README 或在线文档；
  - 要不要暴露原始的 URL；
  - 代理服务承担了大量流量压力。
